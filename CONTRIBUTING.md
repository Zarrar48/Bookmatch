# Contributing Guidelines

Thank you for considering contributing to this project!
Contributions of all kinds are welcome — bug fixes, improvements, and documentation updates.

## Getting Started

1. **Fork the repository**
2. **Clone your fork:**
   ```bash
   git clone [https://github.com/your-username/your-repo-name.git](https://github.com/your-username/your-repo-name.git)
   
## Create a new branch for your changes:

git checkout -b feature-or-fix-name

## Project Structure
- **Backend**: Python (latest stable version)

- **Frontend**: Static HTML files

## Making Changes
- Keep changes small and focused

- Follow clear and readable code style

- Comment your code where necessary

- Do not add unnecessary dependencies

- For frontend changes, use plain HTML only

## Commit Messages
Write clear and meaningful commit messages.

Example:
Fix input validation in backend

## Submitting a Pull Request
1. Push your branch to your fork

2. Open a Pull Request to the main branch

3. Clearly describe:

- What you changed

- Why the change is needed

## Reporting Issues
If you find a bug or have a suggestion:

- Open an issue using the appropriate template

- Provide as much detail as possible

## Code of Conduct
- Be respectful and inclusive

- No harassment or abusive behavior

- Constructive feedback only

By contributing, you agree that your contributions may be used and modified under the project’s license.
